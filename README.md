# Tera - AI Voice Agent Platform

Tera (formerly Dograh) is a comprehensive AI Voice Agent platform designed to build, manage, and simulate intelligent voice assistants for businesses such as hospitality, healthcare, and coaching centers. 

## 🚀 Features

- **Agent Management**: Create, configure, and delete custom AI Voice Agents. Define their system prompts, languages (including Hindi and Hinglish), and workflows.
- **Conversational Chat & Voice Preview**: Test your agent's behavior directly in the browser. Powered by Google Gemini, the chat simulates how the AI will respond based on its system prompt.
- **Dialer Campaign Simulation**: Run outbound campaign simulations. The system uses Gemini AI to generate realistic telephone conversation transcripts between the AI agent and generated customer leads.
- **Call Logs & Analytics**: View realistic call logs, durations, and outcomes generated by the system.
- **Billing Dashboard**: Track usage minutes and estimated costs dynamically based on the call logs.
- **Team Management**: Invite and manage team members and their roles.

## 🛠️ Technology Stack

- **Frontend**: React, Vite, Tailwind CSS, Framer Motion, Recharts
- **Backend**: Node.js, Express.js
- **Database**: Local JSON file store (`db_store.json`) for persistence
- **AI Integration**: Google Gen AI SDK (`@google/genai`) with `gemini-3.5-flash`

## ⚙️ Prerequisites

- Node.js (v18+)
- A Google Gemini API Key

## 🚦 Getting Started

1. **Clone the repository and install dependencies:**
   ```bash
   npm install
   ```

2. **Environment Configuration:**
   Copy the `.env.example` file to create a `.env` file, and insert your Gemini API Key.
   ```bash
   cp .env.example .env
   ```
   *Edit `.env` and set `GEMINI_API_KEY="your-api-key"`*

3. **Start the Development Server:**
   ```bash
   npm run dev
   ```
   The application will start concurrently (Vite frontend + Express backend) and will be accessible at:
   **http://localhost:3000**

## 📖 How it Works

1. **Database Initialization**: On startup, if `db_store.json` does not exist, `server.ts` automatically seeds the database with sample agents (Taj Stay Assistant, Arogya Dental Reception, etc.) and mock invoices.
2. **API Routes**: The Express server provides REST APIs (`/api/agents`, `/api/call-logs`, `/api/billing`, `/api/chat`, `/api/dialer`) to interact with the data.
3. **AI Simulations**: 
   - When using the Chat Preview, the `/api/chat` endpoint constructs a system prompt containing the agent's configuration and conversation history, and queries Gemini for the next response.
   - When using the Dialer, the `/api/dialer` endpoint prompts Gemini to simulate a realistic multi-turn conversation between the configured agent and a lead, parsing the response into call transcripts.
4. **Vite Middleware**: During development, the Express backend uses Vite middleware to serve the React application on the same port, avoiding CORS issues and providing a seamless development experience.

## 📦 Production Build

To build the application for production:
```bash
npm run build
```
This will bundle the Vite frontend into the `dist` folder and compile `server.ts` into a CommonJS bundle (`dist/server.cjs`).

Start the production server:
```bash
npm start
```
